Syntax Breakdown

To hide a subform field from the Main Form, you must iterate through the rows. You cannot simply say Subform.Field.hide() because Deluge needs to know which row you are referring to, even if you want to hide it for all rows.

1

The Iteration Loop

Use for each row in Subform_Link_Name to access the collection of rows.

2

The Row Variable

Inside the loop, use the iterator variable (e.g., row) to target the field: row.Field_Name.hide();

3

The Trigger

Place this script in the On User Input workflow of the field that controls visibility (e.g., the Employment Type dropdown).

Workflow Strategy Analysis

Comparing different methods for controlling visibility based on performance and user experience.

On Form Load

Best for setting initial state based on user role or existing data.

hide Subform.Field;

On User Input

Dynamic changes. Requires row iteration if triggered from Main Form.

row.Field.hide();

On Add Row

Crucial! Ensures logic applies to new rows created after the initial load.

row.Field.show();
